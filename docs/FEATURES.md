# Open Codex CLI Features

This document highlights key capabilities of the **Open Codex CLI**.

- **Zero setup** – install the package and bring your API key.
- **Multiple AI providers** – choose from OpenAI, Gemini, OpenRouter, Ollama or X.AI.
- **Approval modes** – `suggest`, `auto-edit` and `full-auto` for different levels of automation and safety.
- **Sandboxed execution** – runs commands in a network-disabled and directory-constrained environment for security.
- **Multimodal input** – pass screenshots or diagrams to implement features.
- **Memory & project docs** – merge global and project-specific instructions for consistent behavior.
- **Non-interactive / CI mode** – run Codex in automated pipelines.
- **Verbose logging** – set `DEBUG=true` to trace API requests and responses.
- **Recipes** – sample prompts for common tasks like refactoring or generating tests.

For full instructions, installation guides and contribution details, see the main [README](../README.md).
